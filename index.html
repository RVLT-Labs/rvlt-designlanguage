the bit before you ask

Questions, answered.

Why did you build RVLT Flow?+
Because we needed it. Two Toned Productions was juggling audio, lighting, theatre, events, dry hire, full production, crew, venues, quotes, gear movement and compliance. The existing tools either treated us like a simple rental shop or a generic project team. We were neither, so we built the system we kept wishing existed.
Is this proven on real jobs?+
Yes. Flow has been used by our own team from day one. That means the awkward parts showed up early: missing accessories, late quote changes, damaged gear, overbooking risks, venue quirks, compliance gaps and warehouse shortcuts. Those problems shaped the product directly.
How is this different from rental software?+
Rental software usually starts with inventory. Flow starts with the production job. Inventory matters, obviously, but so do labour, suppliers, prep, venue history, compliance, repairs, substitutions, client rates and the operational notes that stop your team learning the same lesson twice.
Can we start small?+
Yes. Start with Core or a single module, then add more as the operation grows. Flow is for teams who want the full connected system from the start.
Can Flow track damaged, missing or unavailable gear?+
Yes. Assets can be marked damaged, missing, in repair, expired, reserved or unavailable. Those statuses affect availability, so broken or non-compliant gear does not accidentally end up on a quote. Revolutionary concept: broken things should not be booked.
Can you migrate our existing inventory?+
Yes. Most teams will start with spreadsheets, exports or whatever the current system allows. RVLT can import equipment, models, categories, kits, clients and job data depending on what you have. The messier the sheet, the less judgement we promise out loud.
